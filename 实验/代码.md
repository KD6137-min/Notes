# argparse

用于优雅地编写命令行参数，定义脚本可以接收哪些命令行参数，自动生成帮助文档 `--help`，自动解析用户输入，把结果转换成 Python 变量，自动做类型检查、默认值、参数必选/可选等

**原理：**运行命令时，shell捕获一个参数列表`sys.argv`传给Python脚本，parser把shell传来的参数列表根据解析器设定的规则解析匹配，并生成Python对象

**基本使用：**

```python
import argparse

parser = argparse.ArgumentParser(
    description="这是一个命令行示例程序",
    formatter_class=argparse.ArgumentDefaultsHelpFormatter # 辅助类，用于美化help输出
)

parser.add_argument("--lr", type=float, default=0.01, help="学习率")
parser.add_argument("--epochs", type=int, default=10, help="训练轮数")

args = parser.parse_args()

print(args.lr, args.epochs) # 自动捕获运行命令的对应参数
```

**核心类：**`ArgumentParser`

`ArgumentParser()` 常用参数：

| 参数名            | 作用                                  |
| ----------------- | ------------------------------------- |
| `description`     | 用于 `--help` 输出的程序描述          |
| `epilog`          | 帮助文档结尾补充说明                  |
| `formatter_class` | 用于控制输出格式（比如换行效果更好）  |
| `add_help`        | 是否自动添加 `-h/--help`（默认 True） |

**核心方法：**`.add_argument()`定义接受规则，解析时按规则解析参数

```python
parser.add_argument(name_or_flags, action, nargs, default, type, required, help)
```

- flag / 开关：使用`action`，值为`"store_true"`(出现为True，不出现为False)或`"store_false"`(出现为False，不出现为True)

  ```python
  parser.add_argument("--use-cuda", action="store_true")
  parser.add_argument("--no-cache", action="store_false")
  ```

- 数值 / 文本：一般是 `--key value` 的形式，使用`type` 指定值类型，`default` 指定默认值

  ```python
  parser.add_argument("--lr", type=float, default=0.001)
  parser.add_argument("--epochs", type=int, default=50)
  parser.add_argument("--name", type=str, default="yolo")
  ```

  - 多值参数：`--key value1 value2 value3`形式，使用`nargs`

    - `nargs='+'`：至少1个
    - `nargs='*'`：0个或多个
    - `nargs='2'`：恰好两个

    ```python
    parser.add_argument("--sizes", nargs="+", type=int)
    ```

  - `type`可接受自定义函数

    ```python
    # 自动校验输入是否合法
    def lr_type(v):
        v = float(v)
        if v <= 0:
            raise argparse.ArgumentTypeError("lr must be > 0")
        return v
    
    parser.add_argument("--lr", type=lr_type)
    ```

- 文件 / 路径：本质上还是字符串，只是表示路径，argparse 本身不会自动检查文件是否存在

  ```python
  parser.add_argument("--output", type=str)
  
  # 可自定义type，如检查文件是否存在：
  def existing_file(path):
      if not os.path.exists(path):
          raise argparse.ArgumentTypeError(f"{path} 不存在")
      return path
  
  parser.add_argument("--config", type=existing_file)
  ```

- 位置参数（无前缀）：必须填，用户不加`--xxx`

  ```python
  parser.add_argument("input")
  parser.add_argument("output")
  ```

  命令必须提供：

  ```shell
  python demo.py input.jpg output.jpg
  ```

- 枚举类：限制参数的合法取值，使用`choices`

  ```python
  parser.add_argument("--mode", choices=["train", "test", "val"])
  ```

- 强制必写参数：使用`required=True`

  ```python
  parser.add_argument("--dataset", required=True)
  ```

**`parser.parse_args()`**：获取所有参数，一次性完成，当脚本执行到 `parse_args()`时才会去查看命令行参数

**help 文档**：写好参数后，你可以直接运行：`python demo.py --help`即可看到自动生成的说明，使用`argparse.ArgumentDefaultsHelpFormatter`可以让`--help`自动显示默认值

```
usage: demo.py [-h] [--lr LR] [--epochs EPOCHS]

这是一个命令行示例程序

optional arguments:
  -h, --help      显示帮助文档
  --lr LR         学习率
  --epochs EPOCHS 训练轮数
```



# ultralytics

`pip install ultralytics`

### 1. **ultralytics YOLOv5**

- **推理（Inference）**：加载一个预训练的模型，进行目标检测、分类等任务

  ```python
  from ultralytics import YOLO
  
  # 加载预训练模型
  model = YOLO('yolov5s.pt')  # yolov5s.pt 是 YOLOv5 的小模型
  # 进行推理
  results = model('image.jpg')  # 输入一张图片进行推理
  
  # 输出推理结果
  results.show()  # 显示检测结果
  results.save()  # 保存推理结果到文件
  ```
  
- **训练模型**：提供训练接口，可使用自定义数据集训练

  ```python
  from ultralytics import YOLO
  
  model = YOLO('yolov5s.pt')
  # 支持在训练时进行各种超参数的调整
  model.train(data='data.yaml', epochs=50, batch=16, lr0=0.01) 
  ```
  
- **评估模型**：

  ```python
  model.val(data='data.yaml')  # 评估模型在验证集上的表现
  ```

- **自动标注（Auto-labeling）**：支持自动标注图片，使用其自带的自动标注工具来生成训练数据

  ```python
  model.export(format='tfrecord')  # 导出为 TFRecord 格式
  ```

#### **常用方法**

- **`predict()`**：用于进行目标检测或分类预测
- **`train()`**：用于训练模型
- **`eval()`**：用于模型评估
- **`export()`**：用于将训练好的模型导出为其他格式（如 `ONNX`，`TensorRT`，`CoreML` 等）
- **`save()`**：用于保存模型权重
- **`show()`**：显示推理结果

## Qwen3-VL 多模态大模型使用

对「图像 + 文本」输入进行对话式推理，生成文本输出

### 核心组件

- `model`：神经网络本体（参数+结构）

  ```python
  model = Qwen3VLForConditionalGeneration.from_pretrained(	# ForConditionalGeneration表示生成模型，支持.generate()
      "Qwen/Qwen3-VL-8B-Instruct",
      dtype="auto",		# 自动选择fp16 / bf16 / fp32
      device_map="auto"	# 自动选GPU，多卡则切层，显存不够则CPU+GPU混合
  )
  ```

  - 方法：`.generate()`，标准自回归生成，输入已有token，输出为继续往后预测token

    ```python
    generated_ids = model.generate(**inputs, max_new_tokens=128)
    # 把prompt部分裁掉，只保留模型生成的回答
    generated_ids_trimmed = [
        out_ids[len(in_ids) :] for in_ids, out_ids in zip(inputs.input_ids, generated_ids)
    ]
    ```

- `processor`：输入/输出的翻译官，功能有：文本 tokenizer、图像预处理、把对话结构整理成模型约定的格式（tensor）、把模型输出的 token转换为人能读的文本

  ```python
  processor = AutoProcessor.from_pretrained("Qwen/Qwen3-VL-8B-Instruct")
  ```

  方法：

  - `.apply_chat_template()`，把 messages 转化为模型规定的 prompt 模板，插入 special tokens，tokenizer → token ids，转成 PyTorch Tensor

    ```python
    inputs = processor.apply_chat_template(
        messages,
        tokenize=True,
        add_generation_prompt=True,
        return_tensors="pt"
    )
    
    # 转变为
    """
    {
      "input_ids": ...,
      "attention_mask": ...,
      "pixel_values": ...   # 图像特征
    }
    """
    ```

  - `.batch_decode()`，把模型生成的 token id 序列 → 转换成人类可读的文本字符串

    ```python
    output_text = processor.batch_decode(
        generated_ids_trimmed, skip_special_tokens=True, clean_up_tokenization_spaces=False
    )
    ```

- `messages`：Qwen 系列定义的**多模态对话协议**，一个`turn`（一个角色在某一时刻的完整发言），包含角色、内容，内容可混合image、text、video

  ```python
  messages = [
      {
          "role": "user",
          "content": [
              {"type": "image", "image": "..."},
              {"type": "text", "text": "Describe this image."},
          ],
      }
  ]
  ```

### 完整流程

```python
from modelscope import Qwen3VLForConditionalGeneration, AutoProcessor

model = Qwen3VLForConditionalGeneration.from_pretrained(
    "Qwen/Qwen3-VL-8B-Instruct", dtype="auto", device_map="auto"
)

processor = AutoProcessor.from_pretrained("Qwen/Qwen3-VL-8B-Instruct")

messages = [
    {
        "role": "user",
        "content": [
            {
                "type": "image",
                "image": "https://qianwen-res.oss-cn-beijing.aliyuncs.com/Qwen-VL/assets/demo.jpeg",
            },
            {"type": "text", "text": "Describe this image."},
        ],
    }
]

# Preparation for inference
inputs = processor.apply_chat_template(
    messages,
    tokenize=True,
    add_generation_prompt=True,
    return_dict=True,
    return_tensors="pt"
)

# Inference: Generation of the output
generated_ids = model.generate(**inputs, max_new_tokens=128)
generated_ids_trimmed = [
    out_ids[len(in_ids) :] for in_ids, out_ids in zip(inputs.input_ids, generated_ids)
]
output_text = processor.batch_decode(
    generated_ids_trimmed, skip_special_tokens=True, clean_up_tokenization_spaces=False
)
print(output_text)
```
