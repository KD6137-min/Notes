# 项目结构

- 数据集与项目分离：数据集统一放～/下，项目目录下放data目录的软链接

  ```bash
  cd Documents/ultralytics
  mkdir -p data
  ln -s ~/datasets/visdrone/detect data/visdrone
  ```

  训练时，命令可用`yolo detect ... data=./data/visdrone/visdrone.yaml`

- 集中存放项目实验输出：`yolo detect ... project=.../ultralytics/runs/detect`，`runs/`放进`.gitignore`，project指定训练结果存放的父目录

- 主动命名实验目录：`yolo detect ... name=xxx`，包含关键信息

- 实验目录内创`note.txt`，记录改动、超参数等

- 项目目录下放`exp_log.md`，记录各个版本改动内容、动机、对照对象、实验结果、结论等



# runs

## weights

- best.pt：保存训练过程中表现 **最好的模型**，通常根据验证集上的某个指标决定，常用于推理 / 部署
- last.pt：保存训练 **最后一轮** 的权重，可用于推理（性能略低于best.pt），resume训练时可作为初始权重



## resume

接上次训练继续训练

两种方式：

- 指定 `model=runs/.../last.pt`：加载模型权重，但 **不会自动加载优化器状态**（比如动量、Adam 的一阶二阶矩等），即继续训练时会重新初始化优化器，学习率调度会从头开始，适合想用已有权重做新训练，不在意优化器状态的情况

  ```python
  yolo detect train data=... model=runs/detect/xxx/weights/last.pt
  ```

- 使用 `resume=True`：加载模型权重（上次训练的last.pt），且 **恢复优化器状态、训练轮数、学习率调度器等**，训练无缝衔接，适合多阶段训练、长时间训练或调参

  ```python
  yolo detect train data=... resume=True epochs=70	# epochs为总轮数
  ```
