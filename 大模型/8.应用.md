# 三、LangGraph 的核心思想（重点）

LangGraph 把整个 Agent 抽象成：

## 1️⃣ 节点（Nodes）

每个节点是一个“步骤”

比如：

- LLM 推理
- 调用工具
- 判断逻辑
- 数据处理

------

## 2️⃣ 边（Edges）

决定下一步走哪

可以是：

- 顺序
- 条件分支
- 循环

------

## 3️⃣ 状态（State）🔥（最重要）

LangGraph 的核心优势：

👉 **所有节点共享一个状态（State）**

类似：

```python
state = {
    "question": "...",
    "thought": "...",
    "tool_result": "...",
    "final_answer": "..."
}
```

每个节点：

- 读取 state
- 修改 state
- 传给下一个节点

👉 这就是“有记忆的 Agent”

# 五、代码结构（给你一点工程感觉）

典型写法：

```python
from langgraph.graph import StateGraph

graph = StateGraph(State)

graph.add_node("llm", llm_fn)
graph.add_node("tool", tool_fn)

graph.add_edge("llm", "tool")
graph.add_edge("tool", "llm")

graph.set_entry_point("llm")

app = graph.compile()
```

执行：

```python
app.invoke({"input": "帮我查天气"})
```

# 六、LangGraph vs LangChain（很关键）

| 对比       | LangChain  | LangGraph        |
| ---------- | ---------- | ---------------- |
| 结构       | 链（线性） | 图（非线性）     |
| 状态       | 弱         | 强（共享 state） |
| 循环       | 难         | 原生支持         |
| Agent 控制 | 黑盒       | 可控             |
| 复杂任务   | 一般       | 很强             |

👉 总结：

> LangChain = 流水线
> LangGraph = 操作系统（Agent runtime）
